# 📸 Screenshot & Video Capture Guide

This guide helps you capture high-quality visuals for your portfolio presentation.

---

## 🎬 Recording Setup

### Recommended Settings

| Setting | Value | Notes |
|---------|-------|-------|
| **Resolution** | 1920x1080 | Full HD minimum |
| **Frame Rate** | 60 FPS | Smooth motion |
| **Bitrate** | 50+ Mbps | High quality |
| **Format** | MP4 (H.264) | Wide compatibility |
| **Audio** | Optional | Background music or narration |

Capture these key scenes for your portfolio:

### 1. Solar System Overview (10-15 seconds)
- **Camera**: Wide view showing all planets
- **Action**: Slow rotation around the sun

### 2. Earth Close-Up (10 seconds)
- **Camera**: Zoomed on Earth

- **Highlight**: Procedural terrain, clouds, atmosphere

### 3. Anti-Gravity Demo (10-15 seconds)
- **Camera**: Medium view of several bodies
- **Action**: Toggle anti-gravity, show repulsion
- **Highlight**: Physics engine capabilities

### 4. Chaos Mode (10 seconds)
- **Camera**: Wide view
- **Action**: Many bodies interacting
- **Highlight**: Performance with 50+ bodies

### 5. UI Showcase (10 seconds)
- **Camera**: Static view with UI visible
- **Action**: Navigate through panels
- **Highlight**: Control Panel, Preset Manager, Debug Panel

### 6. Binary Star System (10 seconds)
- **Camera**: View of both stars
- **Action**: Planets orbiting binary stars
- **Highlight**: Complex orbital dynamics

## 🎨 Composition Tips

### Rule of Thirds
- Position key elements at intersection points
- Leave space in direction of motion

### Lighting
- Ensure planets are well-lit by the sun
- Avoid overexposed areas

### Motion
- Use smooth camera movements
- Avoid rapid pans or zooms
- Let scenes breathe (hold shots for 3-5 seconds)

### UI Visibility
- Ensure text is readable
- Use high contrast themes
- Don't overcrowd the screen

## 📷 Screenshot Capture

### Best Practices

1. **Use highest resolution** your monitor supports
2. **Disable UI** for clean gameplay shots (press Tab)
3. **Enable UI** for feature demonstration shots
4. **Use photo mode** if available (pause simulation)
5. **Capture in PNG** format for lossless quality

### Required Screenshots

| # | Subject | Purpose |
|---|---------|---------|
| 1 | Full solar system | Hero image for README |
| 2 | Earth close-up | Show procedural detail |
| 3 | UI panels visible | Demonstrate UX |
| 4 | Anti-gravity effect | Highlight unique feature |
| 5 | Chaos mode | Show performance |
| 6 | Binary star system | Show variety |

### Annotation Tips

- Use arrows to highlight specific features
- Add brief captions
- Keep annotations minimal and professional
- Use consistent colors and fonts

## 🚀 Quick Workflow

### 1-Hour Capture Session

1. **Setup (5 min)**
   - Configure OBS/recording software
   - Set resolution to 1920x1080
   - Test audio levels (if recording voice)

2. **Capture Raw Footage (30 min)**
   - Record each scene from shot list
   - Do multiple takes if needed
   - Capture extra B-roll footage

3. **Screenshots (10 min)**
   - Pause at key moments
   - Capture 10-15 variations
   - Select best 6-8 later

4. **Review (15 min)**
   - Watch back footage
   - Note timestamps for best takes
   - Plan edit structure
